## How the script works

`bulk_create.py` loops over JSON files in `cases/`, matches each one to an
audio file in `audio/` by file stem (`233333364.json` ↔ `233333364.wav`),
then for each case:

1. Reads `label_template.xml`.
2. For every speaker role with names in the case JSON, generates one
   `<Choices>` block keyed to that role's label.
3. Replaces `{{CHOICES}}` in the template with the generated blocks.
4. POSTs the result as a new Label Studio project's `label_config`.
5. Uploads the matching audio file as one task.

The role mapping lives in the `ROLES` table at the top of `bulk_create.py`:

```python
ROLES = [
    # JSON key,       label value (Arabic),  <Choices> name attribute
    ("judges",        "قاضي",                "judge_name"),
    ("assistants",    "مساعد",               "assistant_name"),
    ("plaintiffs",    "المدعي",              "plaintiff_name"),
    ("defendants",    "المدعى عليه",         "defendant_name"),
    ("lawyers",       "محامي",               "lawyer_name"),
    ("witnesses",     "شاهد",                "witness_name"),
]
```

Contract:
- Every `<Label value="X"/>` in the template that should pull names from the
  case JSON must have a row in `ROLES` whose middle column equals `X`.
- The first column of each `ROLES` row must equal the key under
  `participants.<key>` in the case JSON.
- The template must contain the literal token `{{CHOICES}}` exactly once.
